Qu'est-ce que c'est ?

C'est comme le C/C++

  • Langage compilé vers du code natif
  • Fortement typé
  • Des pointeurs
  • Surcharges d'opérateurs
  • Templates
  • Interfaçage complet avec le C, partiel avec le C++

C'est comme le Java

  • Garbage collector
  • Type référence (plus besoin de pointeurs)
  • Héritage d'une classe, plusieurs interfaces
  • Modules
  • 1 module = 1 fichier -> plus de .h

C'est comme le Python

  • Programmation fonctionelle
  • Fonctions anonymes
  • Fonctions nestées
  • Evaluation paresseuse
  • Typage automatique (inférence)

Et plein d'autres trucs

  • Delegates
  • Programmation par contrat
  • Test unitaires
  • Fonctions @safe, @trusted ou @system
  • Multi plate-forme

Const

Concerne la variable

On peut modifier la valeur d'origine seulement par une autre variable qui n'est pas const

Immutable

Concerne la valeur

La valeur doit pouvoir être mise en ROM; rien ne peut la modifier

{
    assert(sqrt(0) == 0);
    assert(sqrt(1) == 1);
    assert(sqrt(4) == 2);
}
                    

rdmd -unittest -main sqrt.d

@safe, @trusted, @system

  • @safe :
    • Pas d'assembleur
    • Pas d'appel à des fonctions @system
    • Pas de trucs bizarres de façon générale
  • @trusted : Comme @safe, sauf qu'on ne peut pas prouver que la fonction est sûre (bibliothèque C...)
  • @system : tout est autorisé; valeur par défault

La surchage d'opérateurs


class Vector
{
    private float x;
    private float y;

    public Vector opBinary(string op)(Vector other)
    {
        auto result = new Vector();

        result.x = x;
        result.y = y;
        mixin("result.x" ~ op ~ "= other.x;");
        mixin("result.y" ~ op ~ "= other.y;");

        return result;
    }
}
